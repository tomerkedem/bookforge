## נספח א: Quick Reference (מדריך פקודות ותבניות מהיר)

**1. Docker וניהול תשתיות מקומי**

בסביבת הפיתוח המקומית שלנו (Reference Stack 2026), אנחנו מריצים את כלל הרכיבים ב-Docker כדי למנוע את תסמונת "אצלי במחשב זה עובד".

**הפעלה ועצירה של כלל התשתית (Postgres, Kafka, Redis, Chroma):**

Bash

# Start the entire environment in the background (detached mode)

docker compose -f docker/foundation/docker-compose.yml up -d

# Stop and remove the containers (does NOT delete volumes / persisted data)

docker compose -f docker/foundation/docker-compose.yml down

# Remove containers including all persisted data (WARNING: this deletes volumes and data)

docker compose down -v

**בדיקת לוגים של שירות ספציפי (למשל, שרת Kafka):**

bash

docker logs -f aibook_kafka

**3. חוזי נתונים (Pydantic & JSON Schema)**

זהו כלי העבודה העיקרי שלנו לוולידציה סמנטית ומבנית בזמן ריצה.​

**המרת מודל Pydantic ל-JSON Schema תקני (לצורך תיעוד, שמירה ב-DB או תצוגה):**

Python

from my_models import TrafficUpdate  # Import the model we created in Chapter 3

print(TrafficUpdate.model_json_schema())

# Data contract validation error handling and sending to a Dead Letter Queue (DLQ)

data = TrafficUpdate(**raw_json)**

except ValidationError as e:

# 1. Extract detailed error info in a structured format

# 2. Send to the DLQ instead of crashing the system [file:20]

send_to_dlq(payload=raw_json, reason=error_details)

**4. מסד נתונים PostgreSQL והרחבת pgvector**

שימוש נפוץ ויעיל שבו אנחנו ממנפים מסד נתונים יחסית כמאגר הוקטורים (Vector Store) של המערכת.​

**הפעלת הרחבת הווקטורים במסד הנתונים (פעולה חד-פעמית):**

sql

-- Enable the vector extension if it is not already installed

CREATE EXTENSION IF NOT EXISTS vector;

**יצירת טבלה הכוללת עמודת וקטור (לדוגמה, מערך בגודל 1536 ממדים עבור מודלי OpenAI):**

sql

-- Create a table that includes a vector column

-- Example: a 1536-dimension embedding array (commonly used with OpenAI models)

CREATE TABLE document_chunks (

id SERIAL PRIMARY KEY,

content TEXT,

metadata JSONB,

embedding VECTOR(1536)

);

**שאילתת חיפוש היברידי בסיסית (מרחק קוסינוס (Cosine Similarity) בתוספת סינון לפי שדה מטא-דאטה):**

sql

SELECT id, content, 1 - (embedding <=> '[0.1, 0.2, ...]') AS similarity

FROM document_chunks

WHERE metadata->>'category' = 'policy'

ORDER BY embedding <=> '[0.1, 0.2, ...]'

LIMIT 5;



**5. תבניות עיצוב מקוצרות (Architecture Patterns)**

**The Gatekeeper Pattern:**

דפוס שומר הסף.

- העברת כל בקשת מידע חיצונית דרך חוזה נתונים טרם שליחתה למודל ה-AI.

- איתור חריגות, ניתובן ל-DLQ, והחזרת שגיאה אלגנטית (Graceful Degradation) כדי למנוע זיהום של חלון ההקשר (Context) במידע משובש.

**The Safety Sandwich:**

מנגנון עטיפה להפעלת כלים (Tools) על ידי סוכנים חכמים.

- השכבה העליונה מוודאת הרשאות ותקינות קלט לפני הפעולה.

- השכבה האמצעית היא הפעלת הכלי בפועל (למשל פנייה ל-API).

- השכבה התחתונה מוודאת שהפלט מובנה ותקין טרם חזרתו למודל ה-AI, מה שמונע לולאות אינסופיות או קריסות של שירותים.

**Context Budgeting / Truncation:**

אלגוריתם המנהל את "תקציב" הטוקנים.

הוא מקצץ או מסכם חלקים פחות קריטיים (כמו היסטוריית שיחה ישנה) כדי למנוע חריגה ממגבלות המודל ולצמצם את תופעת ה"אבוד באמצע" (Lost in the Middle), שבה מודלים נוטים להתעלם ממידע שנמצא באמצע הטקסט ולזכור רק את ההתחלה והסוף.

## נספח ג: Glossary & Further Reading (מילון מונחים וקריאה נוספת)

**מילון מונחים (Glossary)**

מילון זה נועד ליישר קו בין אנשי הפיתוח, מהנדסי הנתונים וצוותי המוצר, תוך התמקדות בפרקטיקה ההנדסית כפי שהיא מוגדרת ב-2026.

- **Agentic Workflow (תהליך עבודה של סוכן):** מערכת שבה ה-LLM אינו מספק רק טקסט כתשובה, אלא מסוגל לייצר תוכנית פעולה (Plan), להפעיל כלים חיצוניים (Execute) כגון: כתיבה למסד נתונים, ובדיקה של התוצאה (Verify).

- **AI Act (חוק הבינה המלאכותית האירופי):** רגולציה מחייבת מ-2026 המסווגת מערכות AI לפי רמות סיכון. מערכות המשפיעות על החלטות קריטיות (כמו במערכת LogiSmart) מחייבות הוכחת יכולת הסברה (Explainability), בקרת נתונים, ופיקוח אנושי.

- **Context Window (חלון הקשר):** הזיכרון לטווח קצר של המודל. כמות הטוקנים (מילים/חלקי מילים) המקסימלית שהמודל מסוגל לקבל כקלט בשאילתה אחת. ב-2026, קיימים מודלים עם חלון הקשר המגיע למיליוני טוקנים.

- **Data Contract (חוזה נתונים):** הסכם טכני הנאכף בקוד (למשל בעזרת Pydantic), המגדיר לא רק את מבנה הנתונים (Schema) אלא גם את משמעותם העסקית (Semantics) ורמת העדכניות הנדרשת (SLA) לפני כניסתם למערכת ה-AI.

- **DLQ (Dead Letter Queue - תור מכתבים מתים):** תבנית עיצוב שבה נתונים שנכשלו בבדיקת חוזה הנתונים או בעיבוד אינם מפילים את המערכת (Crash), אלא מוסטים לאחסון משני לצורך תחקור מאוחר יותר, תוך שמירה על רציפות הזרמת הנתונים התקינים.

- **Embedding (וקטור ייצוג סמנטי):** ייצוג מתמטי (מערך מספרים/וקטור) של טקסט או תמונה, המאפשר למחשב למדוד קירבה סמנטית (משמעות) בין שני פריטים שונים, במקום להסתמך רק על התאמת מילות מפתח מדוייקת.

- **Eval (Evaluation - הערכה):** תהליך רציף למדידת איכות התשובות של המודל, המבוצע באופן אוטומטי. בניגוד לטסטים מסורתיים שבודקים תוצאה בינארית (עבר/נכשל), Evals מעריכים מדדים רכים יותר כמו מידת ההיצמדות לעובדות (Faithfulness).

- **Golden Dataset:** אוסף קטן ואיכותי של שאלות ותשובות שאומתו על ידי בני אדם, המשמש כבסיס ההשוואה (Ground Truth) למערכות ההערכה (Evals) כדי למדוד האם עדכון למודל פגע באיכות התשובות.

- **Graceful Degradation (קריסה אלגנטית):** עקרון עיצובי שבו כשל ברכיב מסוים מביא לירידה מבוקרת ביכולות המערכת במקום להשבתה מוחלטת (למשל: המרת תשובת בינה מלאכותית מורכבת בתשובה טקסטואלית קבועה בעת שגיאת מערכת).

- **HITL (Human-in-the-Loop - אדם בלולאה):** תבנית בטיחות שבה פעולות בסיכון גבוה שמייצר המודל (כמו עדכון מסלול במסד נתונים) אינן מבוצעות אוטומטית, אלא מושהות עד לקבלת אישור מפורש ממפעיל אנושי.

- **LLMOps:** הסטנדרטיזציה של תפעול מודלי שפה גדולים בסביבת ייצור. כולל את ניהול גרסאות הפרומפטים (Prompt Registry), ניתוב דינאמי של בקשות (Routing), והבטחת איכות וביצועים לאורך זמן.

- **Multimodal RAG:** הרחבה של ארכיטקטורת ה-RAG המסורתית המסוגלת לקלוט, לאנדקס ולחפש לא רק במסמכי טקסט, אלא במקביל גם בתמונות, קול, וידאו או נתוני חיישנים גולמיים.

- **Observability (תצפיתיות):** היכולת להבין את המצב הפנימי של המערכת (בריאות, ביצועים, שימוש במשאבים ועלויות) על בסיס פלט חיצוני, לוגים ומדדים (Metrics). ב-AI, נדרש ניטור גם של איכות התוצאות והזיות.

- **RAG (Retrieval-Augmented Generation):** ארכיטקטורה המשלבת מודל שפה עם מאגר ידע חיצוני. המערכת מחפשת במאגר מידע רלוונטי לשאילתת המשתמש (Retrieval), מצרפת אותו כהקשר נוסף, והמודל מייצר תשובה המבוססת על המידע המעודכן (Generation).

- **Vector Store / DB (מאגר וקטורים):** מסד נתונים המיועד לאחסון ולחיפוש מהיר של וקטורים, כדוגמת pgvector או Chroma. מהווה את הבסיס למערכת השליפה ב-RAG.
